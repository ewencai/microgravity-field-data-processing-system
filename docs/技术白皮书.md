## 1. 概述

微重力勘探对数据处理精度与作业效率要求极高。传统作业模式依赖多个分散的 MATLAB 脚本，中间成果需人工传递、格式各异、易出错且难以质控。本软件将**重力/GPS 数据整理 → 点位匹配 → 多项重力校正 → 质量检查 → 进度统计 → 成果汇总报告**的完整链条集成为单一图形化系统，实现无编程、可追溯、防崩溃的一体化处理。

本白皮书阐述软件的总体架构、核心算法、数据流、关键技术与验证情况。

## 2. 总体架构

### 2.1 架构形态
软件采用 **MATLAB App Designer 单文件类 + 外部功能函数库** 的架构：

```
┌─────────────────────────────────────────────────────┐
│  MicrogravityFieldDataProcessing260428_V22.m（主类） │
│  ├─ properties：UI 组件 + 数据/参数属性              │
│  ├─ createComponents：12 面板 UI 构建                │
│  ├─ callbacks：各面板 运行/保存/加载 回调            │
│  ├─ startupFcn / saveAppState / loadAppState：持久化 │
│  └─ registerPanelRefs / SizeChangedFcn：自适应缩放   │
└─────────────────────────────────────────────────────┘
        │ 调用
        ▼
┌─────────────────────────────────────────────────────┐
│  外部函数库                                          │
│  GravityRawReshape_{CG5,CG6,BurrisDD,BurrisJT}.m     │
│  ReshapeGPSByClass_App.m / GPSPointReshape_App.m ... │
│  地形校正正演 / 报告生成 / 归档拷贝 等               │
└─────────────────────────────────────────────────────┘
```

### 2.2 界面设计规范
统一的布局约定保证一致体验与可维护性：
- 左参数面板 `[1 0 286 671]`，标题居中加粗；右图件面板 `[287 0 1188 671]`。
- 右侧内嵌 `uitabgroup` 采用 `Units='normalized'` 铺满父容器。
- 运行按钮统一绿色 `[0.39 0.83 0.07]` + 白字加粗；保存按钮默认外观。
- 组件"三处同步"铁律：属性声明 + createComponents 赋值 + 引用点，杜绝运行时"属性无法识别"错误。

### 2.3 自适应缩放机制
- 通过 `SizeChangedFcn` 回调 + `AutoResizeChildren='off'` 实现主动缩放控制。
- `registerPanelRefs` 注册各面板参考位置，resize 时按缩放因子 `sx=tgW/1475, sy=(tgH-25)/670` 比例调整。
- 坐标轴与内部 TabGroup 用 normalized 单位填满，避免拉伸变形。

## 3. 核心算法

### 3.1 重力数据分类重塑
- **四仪器差异化解析**：CG5（固定宽度文本，数字 KeyNum 识别）、CG6（空格分隔，字母缩写）、Burris-DD/JT（CSV，20/23 列自适应）。
- **关键词字典分类**：10 类点型（目标/核心/扩展/加密/缺省/静态/动态/联测往/联测返/重力基点），每类含普测与质检。
- **子串匹配防误判**：CG6/Burris 使用 `contains` 模糊匹配，采用 **10→1 反序循环** 确保 `ZJM`（质检）不被 `JM`（加密）子串误覆盖；CG5 使用数值 `==` 精确比较无此问题。

### 3.2 GPS 多格式列映射
- 以 `GPSColMaps.仪器类型.Col` 结构体驱动列映射，支持 E90（41 列，n=3/e=4/h=5，L=7/B=8）与 X10/T7（42 列，含 Role=3，n=4/e=5/h=6，B=7/L=8）。
- 动态遍历 `fieldnames(GPSColMaps)` + `InstrumentType` 匹配，用户可界面扩展新仪器。
- 可选列 `isfield` 守卫访问，保证跨仪器兼容。

### 3.3 重力校正链
按序施加：零漂（线性拟合）→ 固体潮 → 基点差 → 高度（自由空气）→ 纬度 → 中间层（布格，`ρ` 与布格系数可配）→ 地形校正 → 绝对重力 / 准布格异常。格值区分"格值系数"（无量纲）与"格值校正"（mGal）双指标。

### 3.4 地形校正（四步流水线）
1. **加载 DEM**：WGS84，三列 X(经)/Y(纬)/H(高程)。
2. **高程一致性检查**：`scatteredInterpolant` 在测站插值 DEM 高程，计算 `hDiff`；> 50m 警告，> 200m 阻断（垂直基准不一致）。
3. **数据合并**：在 `tolDeg≈11m` 内剔除测站处 DEM 点，将实测点（普测 + 加密）并入 DEM，消除测站高程偏差。
4. **逐点正演**：多面体引力正演 `gravicalc`（G=6.6732e-3），经纬度经 WGS84 大地测量转局部直角坐标，`terrainCorr = -graviz/1000`（负号必需）。

### 3.5 统计量计算
校正与质检面板统一 `computeVectorStats` 输出 18 项：Count/Min/Max/Range/Mean/Median/Std/IQR/百分位(1,5,10,25,75,90,95,99%)/Skewness/Kurtosis。

## 4. 数据流与成果结构

### 4.1 端到端数据流
```
仪器原始文件 ──整理──▶ 结构化成果表(.xlsx/.txt) ──匹配──▶ 匹配表
     └──▶ 各项校正 ──▶ 校正成果 ──▶ 质检/偏移/加密/闭合统计
                                        └──▶ 进度/日报/成果汇总(KPI+图+PPT+日志)
```

### 4.2 标准化成果目录
`预处理结果/工区/` 下 `1.GraDataReshaped` … `12.ProjectSummary` 共 12 级分目录，各面板成果对号入座，保证成果可追溯、可交付。

### 4.3 成果汇总数据源策略（混合）
成果汇总面板采用**内存优先、磁盘回退**策略：优先读取本会话内存结果（`Correction_ResultTable`/`DenseStats_Result`/`QCStats_Result`/`Progress_Result` 等），缺失时回退读取预处理结果目录下已保存的成果表（每项 `exist` 守卫），计算全字段有默认值的 KPI 结构，绝不崩溃。

## 5. 关键技术特性

### 5.1 防御式容错
全流程读表/计算使用 `isfield`/`ismember`/`exist`/`height>0` 守卫，缺失文件/列/空表退默认值；报告生成与目录归档均以 `try/catch` 隔离，单点失败不影响整体。

### 5.2 跨会话持久化
- 通过 `saveAppState`（关闭时）/`loadAppState`（启动时）/`startupFcn`（默认值 + 自动重载）三处同步，将 14+ 路径记忆、各面板可编辑参数、表格数据持久化到 `app_state.mat`。
- 加载时对路径执行 `exist`/`isfile` 回退检查，失效路径保持默认空值。

### 5.3 原始编码保真
GPS(GBK)/重力(ASCII) 原始文件保存时直接 `copyfile`，不做编码转换，避免中文 Windows 工具打开乱码。

### 5.4 高速目录归档
成果归档采用系统 `xcopy`（Windows）绕过 MATLAB 单线程瓶颈，失败回退 `copyfile`。

### 5.5 日期文件夹智能筛选
重力/GPS 数据读取入口仅识别"日期命名的文件夹"（如 `2025.10.01`），通过 `isdir` + `tryMatchDateFormats` 双重过滤，自动忽略项目根目录下的散落文件（参数、文档等）。这使得项目相关文件可直接与数据同放于项目文件夹，便于归档且不干扰数据读取。

### 5.6 默认参数齐全预置
重力/GPS 仪器参数表、基点参数表、校正标量（密度、布格系数、带号、基准面等）均带完整默认值，覆盖常用仪器型号。作业员只需"改值或取消勾选"，无需从空表新增，显著降低首次使用门槛与人为配置错误风险。

### 5.7 参数统一保存
重力参数、GPS 参数、基点与校正参数统一写入单一项目参数文件 `项目参数.mat`（各占独立变量段：`GraPartableData` / `GPSPartableData` / `BaseParams` / `CorrectionParams`），取代以往分散的多个参数文件。三个面板的保存/加载按钮共享同一文件的读写逻辑，便于项目参数的整体管理与迁移。

### 5.8 双格式报告生成（Word + PPT）
- 采用 **"MATLAB 建模 + Python 渲染"** 解耦方案：MATLAB 侧遍历各面板 KPI 与图件，构建结构化 **JSON 文档模型**（段落/表格/图片/样式块），再调用随包分发的内置 Python 由 **python-docx** 渲染 `.docx`、**python-pptx** 渲染 `.pptx`。
- Word（`成果汇总报告_日期.docx`）与 PPT（`成果汇报_日期.pptx`）由两个独立复选框各自控制，可单独或同时生成，互不影响。
- 图件以 `exist` 守卫按需嵌入，缺图优雅跳过；报告生成整体 `try/catch` 隔离，单格式失败不影响 Excel/PNG/归档等其它输出。
- 相较早期基于 MATLAB Report Generator 的 PPT 模板占位填充方案，本方案排版更自由、无需 Report Generator 运行时，且一次建模同时驱动两种格式。

- 校正、匹配等模块均有对应测试脚本与真实工区数据验证。

### 6.3 成果汇总面板验证

- 空态烟测：内存为空直接运行不崩溃，KPI 全 0/'-'，三图空态正常。
- 持久化往返：工区名/版本/归档/Word/PPT 勾选项关-开恢复一致。
- 自适应缩放：窗口缩放时汇总面板按比例联动，无崩溃。
- 报告生成：MATLAB 建模 → Python 渲染，Word（.docx）与 PPT（.pptx）双格式各自独立生成，图件按需嵌入、缺图跳过，输出有效文档。
