## 一、总体部署形态

- 主程序：MATLAB App Designer 单文件类 `MicrogravityFieldDataProcessing260428_V22.m`，经 **MATLAB Compiler** 打包为 Windows 独立 `.exe`。
- 运行支撑：**MATLAB Runtime**（与打包所用 MATLAB 版本一致，本工程为 R2026a）。
- 报告渲染：随包分发的 **Python 报告组件**（python-docx / python-pptx），用于生成 Word/PPT 成果报告；推荐冻结为 `word_gen.exe` / `ppt_gen.exe`，目标机可无 Python。
- 授权：**RSA-2048** 非对称签名授权，机器指纹绑定，纯离线校验。

```
最终交付物
├─ 安装包 微重力野外数据处理软件_Setup_V1.22.2.exe
│   ├─ 主程序 exe + MATLAB Runtime（打包/在线二选一）
│   ├─ 附加资源：Cat.png
│   ├─ 报告组件：word_gen.exe / ppt_gen.exe（或 .py 脚本 + 系统 Python）
│   └─ 桌面快捷方式
└─ 授权：首次启动激活生成 License.key
```

## 四、附加文件清单（必须手工添加）

以下文件通过 `ctfroot` / `system()` 在运行时访问（非 MATLAB 函数调用），**依赖分析器无法自动识别，必须在 Compiler 的"附加文件（Files required for your application to run）"中手工加入**，且随主程序 exe 同目录部署：

| 文件 | 用途 | 是否必需 |
|------|------|---------|
| `Cat.png` | 主窗口图标（`fullfile(pathToMLAPP,'Cat.png')`，V22.m:6107） | 必需 |
| `word_gen.exe` | 冻结的 Word 报告渲染器（优先） | 方案A 必需 |
| `ppt_gen.exe` | 冻结的 PPT 报告渲染器（优先） | 方案A 必需 |
| `build_word_report.py` | Word 渲染脚本（exe 缺失时回退） | 方案B 必需 |
| `build_ppt_report.py` | PPT 渲染脚本（exe 缺失时回退） | 方案B 必需 |

> 报告渲染逻辑（V22.m `runPythonRenderer` / `runPptRenderer`）：
> 先在程序目录找 `word_gen.exe` / `ppt_gen.exe` → 存在则直接调用；否则查找 `build_*.py`，依次尝试 `python3` → `python` 执行。

## 五、Python 报告环境随包配置（两种方案）

### 方案 A（推荐）：冻结为独立 exe，目标机无需 Python
1. 开发机运行 `build_word_gen.bat`（PyInstaller `--onefile --collect-all docx`）生成 `dist\word_gen.exe`，脚本自动拷贝到项目根目录。
2. 同法冻结 PPT 渲染器为 `ppt_gen.exe`（`build_ppt_report.py` + `--collect-all pptx`）。
3. 将 `word_gen.exe`、`ppt_gen.exe` 加入 Compiler 附加文件。
- 优点：客户机零 Python 依赖，启动即用；缺点：需在开发机预先冻结。

### 方案 B：随包分发 Python 脚本 + 目标机装 Python
1. 附加文件加入 `build_word_report.py`、`build_ppt_report.py`。
2. 目标机需安装 **Python 3** 并 `pip install python-docx python-pptx`。
- 优点：脚本可读、便于现场调试；缺点：客户机需配置 Python 运行环境。

> 无论哪种方案，报告生成整体以 `try/catch` 隔离：Python/exe 缺失或失败时仅跳过 Word/PPT，Excel/PNG/归档等其它输出不受影响。

## 六、MATLAB Runtime 选项

打包时（deploytool / `mcc`）选择 Runtime 分发方式：

| 选项 | 说明 | 适用 |
|------|------|------|
| **Runtime included（随包）** | 安装包内嵌 Runtime，离线安装 | 现场无网络（推荐野外部署） |
| **Runtime downloaded（在线）** | 安装时从 MathWorks 下载 Runtime | 有稳定网络，减小安装包体积 |

- Runtime 版本必须与**打包所用 MATLAB 版本严格一致**（R2026a → 对应 Runtime）。
- 目标机首次安装 Runtime 约需数分钟，仅需一次。

## 七、打包步骤（deploytool）

1. 打开工程 `野外微重力数据实时处理系统.prj`（或 `deploytool` 新建 Standalone Desktop App）。
2. 主文件选 `MicrogravityFieldDataProcessing260428_V22.m`。
3. 确认自动分析出的 28 个 .m 依赖；在**附加文件**中加入第四节所列资源（`Cat.png` + 报告渲染器）。
4. 选择 Runtime 分发方式（第六节）。
5. 填写应用名称、版本 `V1.22.2`、图标、发布方信息。
6. 点击 **Package** 生成安装包（`_Setup_V1.22.2.exe`）。
7. 打包后自检：核对生成目录含 `Cat.png`、`word_gen.exe`/`ppt_gen.exe`（或 `.py`）。

> 安全红线（打包前务必核对）：**私钥 `private_key.der`、开发者签发工具 `KeyToolV19.m` / `GenerateKeyPair.m` 严禁进入客户端安装包**；客户端只含 `LicenseManager.m`（内嵌公钥，仅验签）。

## 八、目标机安装与激活

### 8.1 系统要求
- 操作系统：Windows 10 / 11（64 位）
- 内存：建议 8 GB 及以上
- 磁盘：主程序 + Runtime 约需数 GB 空间

### 8.2 安装
1. 双击 `微重力野外数据处理软件_Setup_V1.22.2.exe`，选择安装目录。
2. 如提示安装 **MATLAB Runtime**，按向导完成（首次约数分钟）。
3. 完成后桌面生成快捷方式。

### 8.3 首次启动与激活
1. 双击图标，程序先校验授权。
2. 首次运行弹出**激活框**：将本机**机器码**发给软件提供方。
3. 收到签发的**授权文本**后粘贴激活；`License.key` 自动存于 `%APPDATA%\MicroGravityLicense\`（回退至程序目录），请勿手动修改。
4. 激活通过后进入主界面。

## 九、常见部署问题

| 现象 | 原因与处理 |
|------|-----------|
| 启动报缺 Runtime / 版本不符 | Runtime 未装或版本与打包不一致，重装匹配版本 Runtime |
| 窗口图标缺失 | `Cat.png` 未随包，补入附加文件并置于 exe 同目录 |
| Word/PPT 报告不生成 | 方案A：确认 `word_gen.exe`/`ppt_gen.exe` 在 exe 同目录；方案B：目标机装 Python 3 + python-docx/pptx |
| "未找到授权 / 与本机不匹配 / 已过期" | 按激活流程重新申请；勿回拨系统时钟（内置防回拨） |
| GPS 数据中文乱码 | 软件保持原始 GBK 编码，用中文 Windows 记事本/Excel 打开，勿转 UTF-8 |
| 参数/路径换机丢失 | `app_state.mat` 随机器保存，换机需重设一次后即持久化 |
